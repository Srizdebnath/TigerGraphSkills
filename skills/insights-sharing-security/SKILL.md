# Sharing Dashboards & Securing Access

## When to use this
Use when publishing completed dashboards to executive teams or external users while restricting schema access.

## Prerequisites
Completed dashboard. Links:
- [TigerGraph Insights Docs](https://docs.tigergraph.com)

## Steps
1. Open the target dashboard in Insights.
2. Click the **Share** button in the top menu.
3. Set sharing options: public (unauthenticated link, if allowed by policy) or private (requires login).
4. Assign viewer permissions to specific TigerGraph users or roles.
5. Export the dashboard link or generate embed iframe codes.

## Common mistakes
- Leaving dashboards shared as public when they contain sensitive business data.
- Assuming viewers need admin roles to load dashboards; assign minimal read-only permissions.
